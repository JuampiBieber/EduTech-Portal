# EduTech-Portal - Sprint 1

----------------------------------------------------------------------------------------------------------------------
ROLES DEL EQUIPO:

- Product Owner (PO): define las funcionalidades prioritarias, revisa y aprueba las historias de usuario.
- Scrum Master (SM): guía al equipo en la metodología Scrum y asegura que se siga correctamente el proceso.
- Development Team (DEV): implementa las historias de usuario y realiza los merges, commits y desarrollo en Git.
  
----------------------------------------------------------------------------------------------------------------------
DESCRIPCIÓN DEL SPRINT 1:

Duración: 1 semana.
Objetivo: entregar una primera versión del portal con las HU-01 y HU-02 funcionando correctamente.
Durante el sprint se trabajó con un flujo completo de Scrum, utilizando un tablero Trello con las columnas:
- Product Backlog.
- Sprint Backlog.
- In Progress.
- Review.
- Done.
Cada historia pasó por todas las etapas, con commits, pull requests, revisión del PO y merge final a la rama main.

----------------------------------------------------------------------------------------------------------------------
HISTORIAS DE USUARIO COMPLETADAS:

HU-01 - Página de inicio: 
Como visitante, quiero ver un título del proyecto para identificar la página al ingresar.
Criterio de aceptación: al entrar al sitio se visualiza el título "EduTech Portal".
Estado: completada, revisada y aprobada por el PO.

HU-02 - Lista de cursos:
Como visitante, quiero ver una lista de cursos para conocer el contenido disponible en la plataforma.
Criterio de aceptación: se muestran mínimo tres cursos con su nombre.
Estado: completada, revisada y aprobada por el PO.

----------------------------------------------------------------------------------------------------------------------
FLUJO DE TRABAJO CON GIT:

- Repositorio creado: EduTech-Portal.
- Ramas utilizadas:
  main: rama estable.
  develop: rama de integración.
  feature/HU-01.
  feature/HU-02.
- Cada HU se desarrolló en su rama correspondiente.
- Se realizaron commits descriptivos.
- Se crearon pull requests desde cada rama feature hacia develop.
- Una vez aprobadas por el PO, se hizo merge final de develop a main.

----------------------------------------------------------------------------------------------------------------------

Daily Scrum:
Breve revisión de:

Los cuatro integrantes del grupo nos reunimos diariamente en una casa y trabajamos juntos parte por parte para lograr el objetivo del SPRINT.
  
- ¿Qué voy a hacer hoy?
Seguir trabajando en equipo, planificando el trabajo y analizando cada objetivo para completar el 100% del trabajo en tiempo y forma.
  
- Si tengo algún impedimento.
Por el momento, no.

Review:

- El PO revisó y aprobó ambas historias.
